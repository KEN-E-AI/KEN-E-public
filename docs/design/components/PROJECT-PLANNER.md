# Project Planner

Cross-component tracker for every project PRD under `docs/design/components/`. One row per PRD. The `blocked_by` column lists prerequisite projects so teams can see, at a glance, which projects are free to start in parallel and which are waiting on upstream work.

## How to use this document

- **Delegating work:** A project is ready to start when every entry in its `blocked_by` column is shipped.
- **Parallel execution:** Projects that share the same `blocked_by` set (or have none) can run in parallel by separate dev teams.
- **Adding a new project:** When a new component PRD is authored under `docs/design/components/<component>/projects/`, add a row here using its `<prefix>-PRD-NN` id, the component name, a one-sentence description, and its `blocked_by` entries. Leave `release` blank â€” releases are assigned in a separate planning step.
